## 概览

标准报告采用统一的 8 章节结构，根据报告类型（日报/周报/月报）动态调整章节标题用语。

## 报告类型

| 类型 | 标识 | 默认时间范围 | 输出文件前缀 |
|------|------|--------------|--------------|
| 日报 | `daily` | 当天 | `daily-report-` |
| 周报 | `weekly` | 本周一至周日 | `weekly-report-` |
| 月报 | `monthly` | 本月首日至末日 | `monthly-report-` |

### 章节标题对照

| 章节 | 日报 | 周报 | 月报 |
|------|------|------|------|
| 进展 | 今日进展 | 本周进展 | 本月进展 |
| 问题 | 今日遇到的问题 | 本周遇到的问题 | 本月遇到的问题 |
| 成长 | 今日学习成长 | 本周个人成长 | 本月个人成长 |
| 计划 | 明日计划 | 下周计划 | 下月计划 |

## 结构说明

标准报告包含 8 个主要章节：

```
1. 数据统计
2. {时段}进展
3. 任务完成情况
4. {时段}遇到的问题
5. {时段}个人成长
6. 相关知识分享
7. {下期}计划
8. 风险与问题
```

### 1. 数据统计

**目的**：提供量化的工作数据

**内容**：
- 提交数量
- 参与人数
- 完成的功能点
- 修复的 Bug 数
- 代码量变化

**格式**：
```markdown
| 指标 | 数值 |
|------|------|
| Git 提交数 | 15 |
| 参与人数 | 3 |
| 完成功能 | 4 |
| 修复 Bug | 2 |
```

### 2. 进展

**目的**：概述该时段完成的主要工作

**内容分类**：
- 功能开发（新功能、特性）
- 问题修复（Bug 修复、缺陷修正）
- 技术改进（性能优化、重构、架构调整）
- 文档更新

**格式**：
```markdown
#### 功能开发
- [功能名称] - 简要说明

#### 问题修复
- [Bug描述] - 解决方案

#### 技术改进
- [改进内容] - 改进效果
```

### 3. 任务完成情况

**目的**：展示 Todo/Issue 完成状态

**内容**：已完成与待处理任务列表及统计

### 4. 遇到的问题

**目的**：记录技术难题和解决方案

**内容要素**：
| 要素 | 说明 |
|------|------|
| 问题描述 | 遇到的具体问题 |
| 问题类型 | Bug / 阻塞 / 优化需求 / 技术挑战 |
| 解决方案 | 如何解决 |
| 经验教训 | 学到的经验 |

**格式**：
```markdown
### 问题1: [标题]

**类型**: 技术问题

**描述**: 
问题详细描述...

**解决方案**:
解决方案描述...

**经验教训**:
从中学到的经验...
```

### 5. 个人成长

**目的**：记录个人能力提升

**内容分类**：
| 类别 | 说明 |
|------|------|
| technical | 技术技能提升 |
| soft_skill | 软技能提升 |
| experience | 经验总结 |

**格式**：
```markdown
### [成长点1]

**类别**: 技术技能

**内容**:
成长内容描述...

**影响**:
对工作的影响...
```

### 6. 相关知识分享

**目的**：分享有价值的技术知识

**内容要素**：
| 要素 | 说明 |
|------|------|
| 标题 | 知识点名称 |
| 内容 | 知识详细说明 |
| 链接 | 相关资源链接 |

**格式**：
```markdown
### [知识点名称]

**内容**:
知识点描述...

**相关链接**:
- [链接1](url)
- [链接2](url)
```

### 7. 下期计划

**目的**：规划后续工作（日报为明日、周报为下周、月报为下月）

**内容**：
- 计划完成的功能
- 待解决的问题
- 技术研究目标

**格式**：
```markdown
### 功能开发
- [ ] 计划1
- [ ] 计划2

### 问题修复
- [ ] 修复1
```

### 8. 风险与问题

**目的**：识别潜在风险

**内容**：
| 类型 | 说明 |
|------|------|
| 风险 | 可能影响进度的风险 |
| 阻塞 | 当前阻塞项 |
| 待确认 | 需要确认的事项 |

**格式**：
```markdown
### 风险
- [风险描述] - 影响 - 应对措施

### 阻塞项
- [阻塞描述] - 依赖关系

### 待确认
- [确认事项] - 负责人/时间
```

## 章节规范

### 标题层级
- `#` 一级标题：报告标题
- `##` 二级标题：主要章节
- `###` 三级标题：子章节
- `####` 四级标题：具体条目

### 列表使用
- 无序列表：`-` 或 `*`
- 有序列表：`1.` `2.` `3.`
- 任务列表：`- [ ]` 待完成，`- [x]` 已完成

### 强调语法
- 粗体：`**内容**`
- 斜体：`*内容*`
- 代码：`` `代码` ``
- 链接：`[文字](URL)`
